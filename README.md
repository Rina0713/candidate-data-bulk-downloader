# 👥 候補者情報一括抽出・テキスト自動ビルドツール（ブックマークレット）

## 💻 概要
本ツールは、人材紹介業務における候補者（CAN）情報のリスト化・データ集計プロセスを効率化するために開発した、JavaScript製のブラウザ用スクリプト（ブックマークレット）です。
検索一覧画面に表示された最大50件の候補者データをバックグラウンド処理にて一括で取得・解析し、面談時情報や備考を含む詳細データを一つの構造化されたテキストファイルとして自動生成・ローカルダウンロードします。

## 🎯 開発の背景と解決した課題
従来、自社の営業進捗スレッドや管理スプレッドシートへ移行するための「人材リスト」の作成は、一覧画面から1件ずつ個別の詳細ページを開き、手作業で「学歴」「職種」「年収」「面談コメント」などをコピー＆ペーストして転記する必要があり、日々多大な工数が発生していました。
この組織的な課題を解決するため、「ワンクリックで50名分のコアデータをバックグラウンドで自動スクレイピングし、かつ生成されたデータを独自に構築した専用の生成AI（Geminiエージェント）に投入することで、シートにそのまま貼り付け可能な形式へと自動整形する」一連のワークフローを構築しました。所属部門へ展開した結果、チーム全体のリスト作成および重複チェックに要する時間を劇的に削減し、組織の生産性向上に貢献しました。

## ✨ 技術的なアプローチ・実装ポイント
- **DOM操作による対象データの動的識別：** 一覧表示されたテーブル要素（`.listTable`）の各行から、固有の関数属性（`open_cn_detail`）内に埋め込まれた固有識別子（CANID）および氏名を正規表現で正確に抽出し、処理対象のオブジェクト配列を動的に生成しています。
- **非同期フェッチと複数要素のスクレイピング：** `fetch` APIを用いて詳細画面のHTMLテキストを非同期取得し、`DOMParser`でパース。`input`要素の値（`value`）の抽出、`select`要素の選択済みテキスト（`options[selectedIndex].text`）の取得、特定ID（`COMMENT`など）の要素の網羅など、DOMの構造に応じた確実なデータ抽出ロジックを実装しています。
- **動的な進捗インジケーターによるユーザー体験の向上：** 複数人のデータ取得中にブラウザがフリーズしていると誤認されないよう、画面最前面に固定（`position:fixed`）した進捗用`div`要素を生成。スクレイピングの進捗（現在何人目か・対象者の氏名）をリアルタイムで動的に書き換えるUI/UXを実装しています。
- **Blobオブジェクトを用いたデータポータビリティ：** 抽出したテキストデータをJavaScriptの`Blob`オブジェクトとしてメモリ上に展開し、URL（`URL.createObjectURL`）を動的生成。擬似的なアンカー要素（`<a>`）の作成と自動クリックイベントにより、外部サーバーを介さずブラウザローカルのみで安全にファイルエクスポートを完結させています。
